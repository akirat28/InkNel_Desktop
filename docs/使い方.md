# InkNel 使い方

InkNel は、Markdown でメモを書くためのデスクトップアプリです。ノートは `.md` ファイルとして保存され、画像・添付ファイル・タグ・検索・AI 連携・プラグイン拡張を組み合わせて使えます。

## 基本操作

### ノートを作成する

1. 左端のアクティビティバーで「ファイル」を開きます。
2. サイドバー上部の新規ノートボタンを押します。
3. タイトルと本文を入力します。
4. 本文は自動保存されます。

ノートは 1 件につき 1 つの Markdown ファイルとして保存されます。保存先には `notes/`、画像用の `images/`、添付ファイル用の `attachments/` が作られます。

### フォルダで整理する

- サイドバーの新規フォルダボタンでフォルダを作成できます。
- ノートやフォルダは名称変更できます。
- ノートをフォルダへドラッグすると、そのフォルダへ移動できます。
- フォルダを移動・名称変更した場合、配下のノートとサブフォルダもまとめて更新されます。

InkNel のフォルダは実ファイルのディレクトリではなく、ノートのメタ情報として管理される仮想フォルダです。

### 編集とプレビューを切り替える

ノート上部の切替ボタンで、編集ビューとプレビュービューを切り替えます。

- 編集ビュー: CodeMirror ベースの Markdown エディタ
- プレビュービュー: Markdown を HTML として表示

編集ツールバーを表示している場合、見出し、太字、斜体、打ち消し、コード、リスト、引用、リンク、コードブロック、区切り線、テーブル、日付、テンプレートなどをボタンから挿入できます。

### タブで複数ノートを開く

複数のノートをタブで開けます。設定で「ノートクリック時に常に新しいタブで開く」挙動も選べます。

### 検索する

左端の検索アイコンから検索パネルを開きます。

- タイトルと本文を検索できます。
- 検索結果の一致部分はハイライトされます。
- 検索欄では `↑` / `↓` で過去の検索キーワードを呼び出せます。
- 検索履歴は、起動時にリセットするか、再起動後も保持するかを設定できます。

### タグで分類する

タグパネルでは、ノートに設定したタグごとにノートを一覧できます。タグは本文中の `#word` を自動収集するのではなく、TagBar で明示的に設定したものが対象です。

### 開封履歴を見る

設定で履歴機能を有効にすると、開いたノートの履歴をサイドバーに表示できます。履歴は新しい順に並び、不要になったらクリアできます。

### カレンダーから日付ノートを作る

カレンダー機能を有効にすると、サイドバーにカレンダーを表示できます。

- 日付をクリックすると、その日のノートを作成できます。
- 既に日付ノートがある日は、そのノートを開きます。
- 日付ノートの保存フォルダとタイトル書式は設定で変更できます。
- 日本の祝日やイベント情報が表示される日があります。

## Markdown でできること

### 主な記法

| 用途 | 記法例 |
|---|---|
| 見出し | `# 見出し` |
| 太字 | `**太字**` |
| 斜体 | `*斜体*` |
| 打ち消し | `~~削除~~` |
| インラインコード | `` `code` `` |
| 箇条書き | `- 項目` |
| 番号付きリスト | `1. 項目` |
| タスクリスト | `- [ ] 未完了` / `- [x] 完了` |
| 引用 | `> 引用` |
| リンク | `[表示名](https://example.com)` |
| 画像 | `![代替テキスト](images/xxx.png)` |
| 添付 | `[資料](attachments/xxx.pdf)` |
| テーブル | <code>&#124; 列1 &#124; 列2 &#124;</code> |
| コードブロック | <code>```js</code> |

### コードブロック

コードブロックはシンタックスハイライトできます。対応言語は設定で有効・無効を切り替えられます。

主な対応例:

- JavaScript / TypeScript
- Python
- Go
- Rust
- Bash
- SQL
- YAML
- JSON
- HTML / CSS

コードブロックには、設定によりコピーボタンや行番号を表示できます。

### タスクリスト

`- [ ]` と `- [x]` はチェックボックスとして表示されます。プレビュー上でクリックして状態を切り替えられます。

## 画像と添付ファイル

### 画像を挿入する

画像ファイルをエディタへドラッグ＆ドロップすると、画像が保存され、Markdown に画像リンクが挿入されます。

対応する画像形式:

- PNG
- JPG / JPEG
- GIF
- WebP
- SVG
- AVIF
- BMP

画像は SHA-256 で命名され、同じ画像を複数回追加しても重複保存されにくい仕組みです。プレビュー上の画像はクリックで拡大表示できます。

### 添付ファイルを挿入する

PDF、ZIP、LZH、LHA、7z などをエディタへドラッグ＆ドロップすると、添付リンクとして挿入されます。プレビューでクリックすると OS の既定アプリで開きます。

PDF を追加した場合、1 ページ目のサムネイルを生成できると、画像付きリンクとして挿入されます。

## ノートの保護

ノートには「保護」と「シークレット」を設定できます。

- 保護: 編集時に 4 桁のパスワードが必要になります。保護中のノートは削除できません。
- シークレット: 表示にもパスワードが必要になります。

初期パスワードは `1234` です。設定の「保護」から変更してください。

## テンプレート

テンプレート用フォルダに置いたノートを、現在のノートへ挿入できます。

- 既定のテンプレートフォルダは `template` です。
- 設定でテンプレートフォルダ名を変更できます。
- テンプレートに設定されたタグは、挿入先ノートのタグへマージされます。

## インポートとエクスポート

### Markdown を取り込む

- `.md` ファイルを選択して取り込めます。
- フォルダを選択して、配下の `.md` ファイルを再帰的に取り込めます。

### Markdown / PDF を書き出す

現在のノートを Markdown として書き出せます。印刷または PDF 書き出しも利用できます。

## ゴミ箱

削除したノートはすぐに完全削除されず、ゴミ箱へ移動します。

- ゴミ箱内のノートは復元できます。
- ゴミ箱を空にすると物理削除されます。
- 一定日数以上経過したゴミ箱内ノートを削除する仕組みもあります。

## 保存先と同期

### 保存先を変更する

保存先フォルダを選択できます。保存先を変更すると、`notes/`、`images/`、`attachments/`、`plugins/` がその配下で管理されます。

主な操作:

- 現在の保存先を確認する
- 新しい保存先を選択する
- 現在のデータを新しい保存先へコピーして切り替える
- 保存先の `.md` ファイルから DB を再構築する
- 未参照の画像・添付ファイルを検出して削除する

### ローカル同期

保存先フォルダと SQLite DB の差分をスキャンし、DB からディスクへの書き出し、ディスクから DB への取り込み、削除反映を実行できます。

### クラウド同期

iCloud Drive、Dropbox、Google Drive のローカル同期フォルダを使った同期に対応する設計です。OAuth やクラウド API ではなく、各サービスのローカル同期フォルダへ `InkNel/` データを配置して同期します。

## バックアップとリストア

### バックアップ

設定のバックアップ画面から、保存先フォルダのデータを ZIP として書き出せます。対象は主に次のデータです。

- `notes/`
- `images/`
- `attachments/`
- `plugins/`

### リストア

バックアップ ZIP を選択して復元できます。リストアでは既存の保存先データが置き換えられるため、実行前に確認してください。復元後は Markdown ファイルから DB を再構築します。

## AI 機能

### 対応プロバイダ

設定の AI 画面で、利用する AI プロバイダを選択できます。

- 一般的な AI
- ChatGPT
- ClaudeCode
- Copilot
- Gemini

プロバイダごとに Token、Endpoint、Model、ベースプロンプトを保存できます。

### AI チャット

AI チャットペインでは、現在開いているノートを文脈として AI に質問できます。

できること:

- 現在のノートについて質問する
- 関連ノートを参照して回答を受ける
- チャット履歴を使って会話する
- 進行中の応答を中断する
- 会話を Markdown ノートとして保存する
- PDF または画像を添付して相談する
- URL を含む質問では、アプリ側で URL の本文取得を試み、AI に渡す

### AI 編集モード

AI チャットの編集モードを有効にすると、AI がノート操作を提案・実行できます。

- 新しいノートを作成する
- 現在のノート末尾へ追記する
- 現在のノートを書き換える
- 直前の AI 編集を戻す

破壊的な書き換えは安全ガードにより拒否される場合があります。

### AI 変換

ノート本文に対して、次のような変換を実行できます。

- HTML から Markdown へ変換
- ノート全体の要約
- 本文からタイトル生成
- 見出し別要約
- 箇条書き整理
- テーブル整形
- コードブロック改善
- 関西弁への変換
- 田舎風の口調への変換
- クイズ作成

## プラグイン一覧

InkNel はプラグインで Markdown プレビューやツールバー、サイドバーを拡張できます。プラグインは設定画面のプラグインパネルから取得・インポート・有効化します。

### Mermaid

- ID: `mermaid`
- バージョン: `1.0.3`
- 機能: <code>```mermaid</code> コードブロックを図として描画します。
- できること: シーケンス図、フローチャート、ER 図、ガントチャートなどを Markdown 内に書けます。
- 補足: Mermaid 本体は初回プレビュー時に CDN から動的に読み込まれます。

### Mindmap

- ID: `mindmap`
- バージョン: `1.0.0`
- 機能: <code>```mindmap</code> コードブロックをマインドマップとして描画します。
- できること: ノードのクリック編集、子ノード追加、兄弟ノード追加、削除、ドラッグ＆ドロップでの付け替え、拡大縮小ができます。

### Math (TeX)

- ID: `math`
- バージョン: `1.0.0`
- 機能: <code>```math</code> コードブロックを TeX 記法の数式として描画します。
- できること: Markdown ノート内に数式ブロックを追加できます。
- 補足: KaTeX は初回プレビュー時に CDN から動的に読み込まれます。エディタツールバーから雛形を挿入できます。

### Schedule

- ID: `schedule`
- バージョン: `1.0.0`
- 機能: <code>```schedule</code> コードブロックを 1 日のタイムスケジュール表として描画します。
- できること: ビュー上で予定の開始、終了、内容、場所を追加・編集できます。8 時間表示とスクロール閲覧に対応します。
- 補足: エディタツールバーから雛形を挿入できます。

## プラグインでできること

プラグインは次の拡張ポイントを持てます。

- 特定言語の fenced code block を独自 HTML として描画する
- プレビュー描画後に DOM を加工する
- テーマ変更や再描画時に表示をリセットする
- 設定画面にプラグイン独自の設定 UI を表示する
- アクティビティバーにボタンを追加する
- サイドバーに独自パネルを追加する
- エディタツールバーにボタンを追加する

## プラグインの取得と有効化

1. 設定を開きます。
2. 「プラグイン」を選びます。
3. プラグインカタログを取得します。
4. 使いたいプラグインをダウンロードします。
5. インストール済み欄でインポートします。
6. トグルを ON にして有効化します。

ダウンロードはファイル保存、インポートは実行時レジストリへの登録です。ダウンロードしただけでは有効化されません。

## プラグインの保存先

プラグインは保存先フォルダ配下またはアプリのユーザーデータ配下の `plugins/` に保存されます。各プラグインは manifest と本体 JavaScript ファイルで構成されます。

例:

```text
plugins/
├── mermaid.json
├── mermaid.js
├── mindmap.json
├── mindmap.js
├── math.json
├── math.js
├── schedule.json
└── schedule.js
```

## 開発者向けプラグインメモ

プラグインは ES module の JavaScript と manifest で構成されます。主な export は次の通りです。

```js
export const manifest = {
  id: 'example',
  label: 'Example',
  description: '説明',
};

export const renderFence = ({ code, lang, escapeHtml }) => {
  return null;
};

export const renderInPreview = async (root, ctx) => {
  // プレビュー DOM を加工
};

export const resetInPreview = (root) => {
  // 再描画前の後始末
};
```

開発モードでは `plugin-dev/plugins/` のカタログとファイルを直接配信できます。これにより、ダウンロードやインポートを省いてプラグインを試作できます。

## データの扱い

InkNel の主要データは次の場所に分かれます。

- SQLite DB: ノートメタ情報、フォルダ、タグ、設定、検索用情報など
- `notes/`: ノート本文の Markdown ファイル
- `images/`: ノートに挿入した画像
- `attachments/`: PDF や ZIP などの添付ファイル
- `plugins/`: ダウンロードしたプラグイン

ノート本文は Markdown ファイルとして保存されるため、他の Markdown エディタでも読みやすい形式です。一方、フォルダ、保護状態、タグなどのメタ情報は DB と front matter によって管理されます。
